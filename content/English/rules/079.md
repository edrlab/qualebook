abstract: "The EPUB Navigation Document (nav.xhtml) is a specialized XHTML file providing essential navigation features, such as the table of contents, enabling quick and easy access to content. This document must comply with EPUB 3.3 specifications and include, at a minimum, a structured table of contents."

Meo: 
- "Create a file named `nav.xhtml` in the EPUB content folder."
- "Declare this file in the OPF package manifest using the `properties=\"nav\"` attribute."
- "Include at least one `<nav epub:type=\"toc\">` element containing the main table of contents."
- "Structure the table of contents using an `<ol>` (ordered list) element containing `<li>` elements for each entry."
- "Use `<a>` links pointing to the corresponding sections of the ebook."
- "Add a heading to the navigation section (e.g., `<h1>Table of Contents</h1>` or `<h2>Contents</h2>`)."
- "Create a hierarchy of entries by nesting `<ol>` lists for subsections."
- "Optionally, add other navigation types: `<nav epub:type=\"landmarks\">` for landmarks, `<nav epub:type=\"page-list\">` for the page list, `<nav epub:type=\"loi\">` for the list of illustrations, `<nav epub:type=\"lot\">` for the list of tables."
- "Consider adding `epub:type=\"frontmatter\"` to the `<body>` element of the navigation document."
Controle:
- "Test navigation across different reading systems. Ensure that all table of contents links are functional and target the correct sections."
- "Verify that the heading hierarchy in the table of contents accurately reflects the ebook's structure."
- "Confirm that the order of entries in the table of contents corresponds to the logical reading order of the ebook."
